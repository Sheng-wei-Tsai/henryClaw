# Repository Memory

## Stable Context
- 🦞 這個 repository 目前可確認的長期脈絡非常少；近一段時間的 issue 記憶幾乎只反覆指向 **#1**，而它已被一致判定為**沒有實質內容的占位／測試性 issue**，且已於 **2026-05-15** 關閉。
- 🦞 龍蝦群已形成穩定共識：**關閉的空白 issue 不應再被視為活躍工作的一部分**，也不應從標題或編號自行腦補需求。
- 🦞 目前看不出穩定的產品方向、技術主題、協作模式或生活節奏線索；這不是因為一切都已完成，而是因為**可用記憶資料不足**，無法安全推導。
- 🦞 對這個 repo 來說，現階段較穩定的不是功能上下文，而是**記憶治理原則**：先確認 issue 是否有足夠背景，再決定是否納入長期追蹤。

## Recent Themes
- 🦞 最近幾天的主題幾乎都不是功能開發，而是**記憶清淤、任務盤點與 backlog 去噪音**。
- 🦞 龍蝦群反覆在做同一件事：確認 **#1** 沒有需求描述、沒有討論脈絡、沒有實作紀錄，因此將它從有效工作脈絡中排除，避免主人之後誤以為還有一條未完成支線。
- 🦞 另一個近期反覆出現的訊號是：**issue 資訊品質直接決定可接手性**。只要缺少目標、背景、驗收條件或來源對話，龍蝦們就無法可靠延續工作。
- 🦞 近期資料沒有顯示出其他明確進行中的工程、除錯、產品決策或生活安排；若主人其實有推進其他事情，目前 snapshots **沒有足夠證據反映**。

## Constraints
- 🦞 **不可憑空補需求**：若 issue 缺少明確目標、背景、驗收條件或討論來源，龍蝦群應停止推論，不把它當成可執行任務。
- 🦞 **issue 關閉狀態可作為歸檔依據**：像 #1 這類已關閉且內容空白的項目，應視為歷史殘影，而非待辦。
- 🦞 **外部脈絡缺失會中斷記憶鏈**：若原始背景來自 Telegram 或其他外部對話，但沒有連結或摘要，後續 agents 幾乎無法接手。
- 🦞 **目前資料覆蓋率偏低**：近 30 天 snapshots 幾乎沒有提供 #1 以外的有效 issue 脈絡，因此任何「repo 正在做什麼」的結論都帶有高度不確定性。
- 🦞 shared manual notes 目前只有流程說明，**沒有額外的人類手動長期記憶可供補強**。

## Open Loops
- 🦞 若主人想重啟 **#1** 相關方向，仍需補齊至少以下資訊：
  - 明確目標
  - 背景脈絡
  - 預期成果或驗收條件
  - 若有外部討論，需附連結或摘要
- 🦞 目前不確定主人是否有其他未被 snapshots 捕捉到的工作流；若有，後續需要補進更多 issue 記錄或人工筆記，否則龍蝦群只能記住「資料不足」而不是實際進展。
- 🦞 尚未建立可重用的 repo 任務模板或 issue 最低資訊標準；雖然近期記憶已反覆顯示這是痛點，但還看不到主人是否已正式採納某套流程。
- 🦞 除了 #1 的歸檔結論外，**目前沒有可確認的活躍工程 open loop**；這個空白本身也應被視為一種不確定性，而不是進度健康的證明。
