# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定脈絡很少：這個 repo 的記憶整理流程依賴 **GitHub issue / comment** 作為原始資料來源，長期記憶應該是從這些來源蒸餾出的結論，而不是逐字複製日誌。
- `.memory/shared/manual.md` 是給人類手動維護的長期記憶區，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- compact-memory workflow 會讀取手動筆記，但**不會覆寫**手動筆記；因此人工維護內容應視為可持續保留的高優先級背景。
- 目前提供的 daily snapshots 中，沒有任何可用 issue 內容，因此**尚無法可靠判定**此 repo 的產品目標、技術棧、工作流程偏好或主人近期實際在處理的任務。

## Recent Themes
- 最近一整段整理視窗（2026-04-30 至 2026-05-06）都呈現相同狀態：**沒有可用 issue**。
- 因為缺乏活躍 issue，龍蝦群這段時間的主要行動不是新增記憶，而是**保留既有記憶、等待新訊號**。
- 目前沒有可辨識的跨 issue 主題，也沒有新的跨 issue 決策可蒸餾成長期規則。

## Constraints
- 記憶整理必須以 **curated long-term memory** 為目標，不能把 MEMORY.md 寫成：
  - 原始 issue 日誌
  - 每日摘要的逐段拼貼
  - 單純索引頁
- 若某件事只在單日短暫出現，且尚未證明是穩定事實，應放在「近期主題」或「未完成事項」，**不能過早提升為穩定規則**。
- 當資訊不足或彼此矛盾時，必須明確標示不確定性；以目前資料量來看，這是常態而不是例外。
- 目前資料來源高度稀少，代表任何關於 repo 習慣、主人偏好、技術決策的推論都應保守處理。

## Open Loops
- 等待下一輪 issue 更新後，再判斷是否有新的長期記憶可蒸餾。
- 需要未來的 issue / comment 來補足以下關鍵空白：
  - repo 的實際用途與主要維護目標
  - 主人常見的工作模式與偏好
  - 穩定的工程規範、限制與決策
  - 目前真正未完成的重要事項
- 在新資料出現前，龍蝦群只能確認：**目前沒有足夠證據形成新的長期記憶**。
