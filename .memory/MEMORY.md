# Repository Memory

## Stable Context
- 龍蝦群目前能穩定確認的 repo 脈絡很少；`Sheng-wei-Tsai/henryClaw` 在近一週觀測中，幾乎只出現 **Issue #1** 這個工作入口。

- 這個 repo 目前較像是**協作記憶基礎設施**，而不是已有明確產品主線的開發現場。龍蝦群反覆觀察到的重點，是先保留入口、等待後續脈絡補齊。
- 可推定的一項工作偏好是：**希望把後續協作內容整理成可接手的濃縮記憶，而非逐字聊天存檔**。這點在多日快照中反覆出現，已可視為目前較穩定的整理方向。
- 目前**沒有足夠證據**支持任何具體產品方向、技術架構、模組邊界或開發優先序；若要記住這個 repo 的長期事實，最準確的說法是：**它的記憶入口已建立，但任務語意尚未成形**。

## Constraints
- **不可臆測需求**：在缺少 issue 內文、留言、外部對話與驗收條件時，龍蝦群應避免自行定義工作範圍、優先級或解法。
- **以原始 issue / comment 為準**：手動筆記只是長期記憶的整理層，不應取代 GitHub issue 與 comment 作為原始事實來源。
- **記憶應濃縮，不應逐段複製原文**：這個 repo 的記憶整理目標是可重用、可交接的 context，而不是聊天逐字稿或日誌索引。
- **目前不適合估工、排程或拆任務**：因為 #1 的目標、背景、預期成果與完成條件都尚未明確。
- **可追溯性受限**：若沒有 Telegram 對話或其他補充紀錄，很多決策歷史無法可靠重建，只能誠實標記為未知。
- **資料樣本過少**：近幾日幾乎只有單一 issue 可觀察，因此任何「repo 慣例」判斷都應保守；除了記憶整理偏好外，其他規則都還不夠穩定。
