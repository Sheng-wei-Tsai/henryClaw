# Repository Memory

## Stable Context
- 龍蝦群目前能確認的長期脈絡很少：這個 repo 的記憶來源以 **GitHub issue / comment** 為原始事實基礎，memory 內容應該是整理後的長期脈絡，而不是逐字日誌。
- `.memory/shared/manual.md` 是給人類手動維護的長期記憶區，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- compact-memory workflow 會讀取手動筆記，但**不會覆寫**它；因此手動筆記可視為較穩定的人類維護層。
- 目前提供的近日日報中，**沒有可用 issue 活動**，所以尚無法從近期資料提煉出更具體的專案領域知識、工作流程偏好或技術決策。

## Recent Themes
- 最近一段時間（2026-05-07 至 2026-05-13）的共同主題非常一致：**沒有新的 issue 資料可整理**。
- 每日 snapshot 都顯示：
  - 沒有 agent 可彙整的 issue 活動
  - 沒有可辨識的跨 issue 主題
  - 沒有新的跨 issue 決策
- 因此，近期狀態比較像是**記憶維護空窗期**，而不是專案進入了某個明確的新方向。
- 不確定性：這代表的是 repo 近期真的沒有工作流動，還是資料視窗／抓取條件下剛好沒有可見 issue，現有資訊不足以判定。

## Constraints
- 記憶整理必須以 **curated long-term memory** 為目標，避免把 daily snapshot 或 issue 內容原樣堆疊進來。
- 原始資料權威來源仍是 **GitHub issue / comment**；memory 檔案屬於摘要與蒸餾層，不應取代原始紀錄。
- 若某件事只在單日短暫出現、尚未形成穩定模式，應放在「近期主題」或「未完成事項」，**不要誤寫成長期規則**。
- 目前資料量不足，任何關於：
  - repo 技術棧
  - 主人的工作習慣
  - 穩定開發流程
  - 長期待辦方向  
  都不能做強推論；龍蝦群應維持保守記錄。
