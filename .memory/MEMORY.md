# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定脈絡很少：這個 repo 的記憶整理流程依賴 **GitHub issue / comment** 作為原始資料來源，人工維護的 shared notes 只用來保存長期記憶與共同規則，不應直接複製原始 issue 內容。
- `.memory/shared/manual.md` 是給人類手動維護的長期記憶區；compact-memory workflow 會讀取它，但**不會覆寫**它。
- 目前可見資料顯示，這份記憶系統的目標是把零散 issue 訊息蒸餾成可重用的長期 context，而不是做日誌備份或索引。

## Recent Themes
- 最近一整段整理視窗（2026-05-01 至 2026-05-07）**都沒有可用 issue**，因此沒有新的工作脈絡、生活習慣、決策或跨議題主題可供提煉。
- 這段期間的主旋律是：**維持既有記憶，不新增推論**。龍蝦群目前只能守著空網，等待新的 issue 活動浮上來。
- 由於連續多日都沒有資料，現階段不適合把任何新事項寫成穩定規則；若未來出現新訊號，應先放入近期主題，再觀察是否反覆出現。

## Constraints
- 原始事實來源以 **GitHub issue / comment** 為準；記憶文件只能做整理與蒸餾，不能取代原始紀錄。
- 不應把 daily snapshots 逐段複製進長期記憶；應優先抽取可重用、跨日仍成立的 context。
- 若資訊不足、沒有 issue、或訊號彼此矛盾，必須明確標示不確定性，避免把短暫現象誤寫成長期事實。
- 目前資料量不足，**不能可靠推定** repo 的實際工作內容、主人習慣、技術棧或長期決策。

## Open Loops
- 等待下一輪有實質內容的 issue / comment 出現，才能恢復正常的長期記憶整理。
- 需要未來資料來回答的關鍵空白包括：
  - repo 的主要用途與工作領域是什麼；
  - 主人有哪些穩定的開發習慣、偏好與限制；
  - 是否存在長期未完成事項、固定流程或重要決策。
- 在新資料出現前，龍蝦群應維持保守策略：**保留既有框架，不憑空補完脈絡**。
