## Constraints

- 在 `#1` 缺乏明確描述前，**不能把它視為可直接開工的工程 issue**。
- 目前 repo 記憶高度依賴 issue 內容；若主人把討論留在 Telegram 或其他外部管道、卻沒有回填，龍蝦群就無法可靠延續記憶鏈。
- 由於近 30 天內可見資料極少，任何關於優先順序、責任分工、技術方案、完成標準的判斷，都有高不確定性。
- 目前沒有 labels、沒有穩定的 issue 群組、也沒有足夠多的工作樣本可歸納 repo 慣例；若未來資料增加，這份記憶需要重新校正。
