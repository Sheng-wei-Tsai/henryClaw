# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定脈絡很少：這個 repo 的記憶整理流程會從 `.memory` 讀取人工維護筆記與每日快照，目標是替主人保留可長期重用的工作脈絡，而不是保存原始 issue 日誌。
- `shared/manual.md` 被定位為**人類手動維護的長期記憶區**，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- 原始事實來源仍以 **GitHub issue / comment** 為準；記憶文件應做整理與蒸餾，不應整段複製原文。
- compact-memory workflow 會讀取手動筆記，但**不會覆寫**它；表示人工補充的長期記憶具有保留價值。

## Recent Themes
- 最近一段時間（至少 2026-06-15 到 2026-06-21）的 daily snapshots 都顯示：**沒有可用 issue 可供整理**。
- 因為缺乏新的 issue 活動，目前**沒有可辨識的跨議題主題**，也沒有新的跨 issue 決策浮現。
- 近期主旋律不是功能演進，而是：**維持既有記憶、等待新的工作訊號出現**。

## Constraints
- 記憶整理必須以**長期可重用**為優先，避免把 daily snapshot 當成逐日備份或索引頁。
- 若某件事只短暫出現、尚未形成穩定模式，應放在近期主題或 open loop，而**不應寫成穩定規則**。
- 目前資料量不足，無法可靠推定：
  - 主人的開發習慣
  - repo 的技術棧偏好
  - 常見工作流程
  - 已知產品方向或長期 roadmap
- 現有 daily snapshots 高度一致且資訊稀少，因此任何更具體的結論都會過度推論；龍蝦群應保持保守。

## Open Loops
- 等待下一輪有實質內容的 GitHub issue / comment 出現，再補強這份長期記憶。
- 目前缺少可驗證的 repo 工作脈絡；之後若有新議題，優先觀察並沉澱：
  - 是否存在穩定的開發規則
  - 是否有反覆出現的技術主題
  - 是否有主人明確表達的限制、偏好或禁忌
- `shared/manual.md` 目前只有使用說明，**尚未看到人類手動補入的具體長期記憶**；若主人有已知規則或背景，值得補寫進去。
