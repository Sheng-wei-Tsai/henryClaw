# Repository Memory

## Stable Context
- 龍蝦群目前沒有從最近一段時間的 issue 視窗中撈到可用內容，因此**沒有足夠證據建立新的長期穩定規則**。
- 這個 repo 的長期記憶維護方式很明確：
  - **GitHub issue / comment 是原始事實來源**。
  - `.memory/shared/manual.md` 用來保存**人工維護的長期記憶**，適合放穩定規則、長期決策、常見限制與 agent 共通習慣。
  - compact-memory workflow 會讀取 manual notes，但**不應覆寫人工筆記**。
- 目前可確認的 repo context 很有限；除了記憶維護流程本身之外，**尚無足夠資料可推定專案目標、技術棧、工作節奏或主人偏好**。

## Recent Themes
- 最近多日（2026-04-25 至 2026-05-01）的 daily snapshots 都呈現同一個訊號：
  - 整理視窗內**沒有可用 issue**。
  - **沒有可辨識的跨 issue 主題**。
  - **沒有新的跨 issue 決策**。
- 因此近期主題不是某個功能或專案方向，而是：
  - 記憶系統處於**低活動／無新輸入**狀態。
  - 龍蝦群目前只能**保留既有記憶框架**，等待新的 issue 活動提供可蒸餾的長期脈絡。

## Constraints
- 長期記憶整理必須遵守以下限制：
  - **不要把 issue 原文整段複製進記憶**；應該蒸餾成可重用的 repo context。
  - **只有反覆出現、足夠穩定的資訊**才應提升到長期記憶。
  - **只在最新一天短暫出現、尚未穩定的內容**，應留在近期主題或 open loop，不應寫成固定規則。
  - 若資訊不足或彼此矛盾，必須**明確標示不確定性**。
- 目前最大的實際限制是：
  - 最近連續多天**沒有 issue 資料可供整理**，所以無法可靠推導主人近期工作內容、未完成功能、協作模式或偏好變化。

## Open Loops
- 等待下一輪有實質內容的 GitHub issue / comment 出現，讓龍蝦群能重新建立：
  - 專案的穩定工作主題
  - 主人的長期偏好與決策模式
  - 常見限制與 repo 慣例
  - 真正需要追蹤的未完成事項
- 目前有一個明確的不確定性：
  - **不清楚是專案近期沒有活動，還是整理視窗未能取得資料**；在有新證據前，不應做進一步假設。
- 在新的 issue 訊號出現前，`MEMORY.md` 應維持保守，不虛構 repo context。
