# Repository Memory

## Stable Context
- 龍蝦群目前能確認的長期事實很少：這個 repository 在最近可見的 issue 記憶中，**幾乎沒有形成穩定的產品線、功能線或工程主線**。
- 唯一反覆出現的條目是 **Issue #1**，但它已被一致判定為：
  - 早期占位／測試性質的空白 issue
  - 缺乏需求描述、背景、討論、驗收條件與實作脈絡
  - **不應視為有效待辦或可延續任務**
- 龍蝦們已形成可重用的判準：**若 issue 沒有明確目標、背景與可追溯脈絡，就不應自行腦補需求並展開執行。**
- 目前沒有足夠證據顯示主人已在 repo 內建立穩定的 issue 驅動工作流程；反而較明顯的是，**記錄品質會直接影響後續接手與記憶整理能力**。

## Recent Themes
- 近期主題不是功能開發，而是**記憶清點、任務除草、降低 backlog 噪音**。
- 多日快照反覆指出同一件事：龍蝦群花的力氣主要在確認 **#1 已無需追蹤**，避免把空白卡片誤認為仍在進行中的工作。
- 近期也反覆浮現一個流程訊號：**repo 內至少出現過「先開 issue、但沒有補齊內容」的情況**。這讓 daily snapshot 難以還原主人的真實工作／生活脈絡。
- 因資料過薄，近期**看不出穩定的技術方向、協作節奏、產品決策或生活安排模式**；這不是代表沒有事情發生，而是目前記憶來源不足以支持判定。

## Constraints
- **資料來源限制明顯**：目前近 30 天可整理出的 issue 幾乎只有 #1，且內容空白，因此很多結論只能停留在「資訊不足」。
- **不可從空白 issue 推導需求**：沒有目標、背景、討論或驗收條件時，龍蝦群應保守處理，不虛構工作內容。
- **Issue 才是主要可追溯記憶節點**；若工作沒有進入 issue，或 issue 沒寫清楚，後續就難以接手、整理與延續。
- 若未來希望 daily memory snapshot 更有用，新的 issue 至少應補上：
  - 任務目的／問題定義
  - 背景脈絡
  - 預期成果或驗收條件
  - 若有外部對話（如 Telegram），應附摘要或連結
- 目前**沒有足夠資訊**確認 repo 的其他工程慣例、技術棧、部署方式、分工模式或固定工作節奏。

## Open Loops
- **#1 的原始意圖仍不明**：若主人當初其實想追某件事，現在需要重新開一張有內容的新 issue，而不是依賴舊的空白卡片。
- **是否存在未被 issue 化的工作仍不確定**：目前看不到，不代表不存在；只是龍蝦群沒有足夠殼內證據可記。
- **記錄流程仍待補強**：若主人希望這個 repo 能累積真正可用的長期記憶，後續需要提高 issue 的上下文完整性。
- 目前**沒有可確認的技術 blocker、未完成功能或待驗證方案**；但這個結論帶有高度不確定性，主因是資料不足，而不是事情真的都已收束。
