# Repository Memory

## Stable Context
- 龍蝦群目前能穩定確認的 repo 脈絡很少；這個記憶池在近一段時間內**沒有形成明確的產品開發主線、技術主題或多 issue 協作節奏**。
- 唯一反覆被確認的既有事項是 **Issue #1 已於 2026-05-15 關閉，且應視為占位／測試性節點**，不是有實質需求的工作項目。
- 對這個 repo 的長期記憶整理，應採取**保守歸檔**原則：  
  - 沒有需求描述、背景脈絡、成功條件或討論紀錄的 issue，**不應自動視為有效待辦**。  
  - 對資訊不足的項目，應先標記為不可執行或待補件，**不要腦補任務內容**。
- 目前看來，龍蝦群在這個 repo 的主要價值偏向**記憶衛生與上下文除噪**，而不是承接一條已成熟的工程流水線。

## Recent Themes
- 最近數日反覆出現的主題不是功能開發，而是**issue hygiene／任務衛生整理**：把空白、無上下文、不可執行的節點明確歸類，避免 backlog 被雜訊污染。
- **#1 被持續確認為可略過項目**：  
  - 無需求描述  
  - 無留言或討論脈絡  
  - 無實作進度或驗收資訊  
  - 已關閉且沒有證據顯示應繼續追蹤
- 龍蝦們近期反覆守住一條紀律：**資料不足時要誠實寫不足，不要把空白 issue 包裝成進展**。
- 近期也隱約暴露出一個可能的工作流問題：**部分工作脈絡可能散落在 repo 外**（例如聊天或其他外部來源），導致回頭整理時難以重建上下文；但目前證據不足，這只能列為傾向，不宜寫成既定事實。
- 整體而言，最近這個 repo 比較像在做**清點、封存、排除誤導訊號**，而不是推進新功能、修 bug 或做架構決策。

## Constraints
- **資訊密度極低**：近 30 天可見記憶幾乎只圍繞 #1，無法可靠推導更大的專案方向。
- **缺乏原始上下文**：目前看不到足夠的 issue 討論、PR、commit、外部對話或驗收紀錄，很多事情只能保守判讀。
- **不可把空白 issue 當成需求來源**：若沒有目標、背景、範圍與完成條件，龍蝦群無法安全展開拆解或執行。
- **不確定性需明講**：目前無法確認  
  - 是否存在 repo 外的實際工作主線  
  - 主人是否有其他未 issue 化的工作  
  - #1 是否曾有外部討論但未被納入記憶
- 因此，這份記憶文件應維持**低假設、高可信度**；寧可留白，也不要製造假的長期脈絡。

## Open Loops
- **若未來要重啟或重用 #1**，主人需要先補齊至少以下資訊：  
  - 目標是什麼  
  - 背景脈絡是什麼  
  - 想解決的問題是什麼  
  - 預期產出或完成條件是什麼
- **Issue 建立流程是否需要更明確的最小欄位**，目前是個未解問題。近期反覆出現的空白 issue 現象顯示：若沒有基本需求模板，龍蝦群難以接手。
- **是否有 repo 外遺失的上下文** 仍不確定；若主人之後補上聊天紀錄、需求說明或其他來源，龍蝦群才有機會重建更完整的長期記憶。
- 除了上述事項外，**目前沒有可確認的其他未完成工程任務**；若實際上有主線工作，現有記憶資料尚不足以辨識。
