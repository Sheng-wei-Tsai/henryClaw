# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定脈絡很少：這個 repo 的長期記憶來源分成兩層，**GitHub issue / comment 是原始事實來源**，而 memory 檔案只負責整理、蒸餾與保留可重用脈絡。
- `shared/manual.md` 是給人類手動維護的長期記憶區，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- 目前提供的 daily snapshots 在連續多天內都沒有可用 issue，因此**尚無法從近期活動中提煉出更具體的 repo 工作模式、技術決策或主人偏好**。
- 可合理視為目前狀態的是：這份記憶應保持「**curated long-term memory**」定位，而不是日誌、逐日摘要或 issue 索引。

## Recent Themes
- 最近一段時間（至少 2026-05-06 到 2026-05-12），daily snapshots 一致顯示：
  - 沒有可用 issue
  - 沒有可辨識的跨 issue 主題
  - 沒有新的跨 issue 決策
- 因此近期主題不是某個功能或專案方向，而是**資訊真空期**：龍蝦群暫時只能守住既有記憶，等待新的 issue 活動出現。
- 目前沒有足夠證據判斷這是短暫空窗、repo 低活躍期，還是資料收集流程尚未接到事件；此處存在不確定性。

## Constraints
- 記憶整理必須遵守以下限制：
  - **不要把 issue 原文整段複製進記憶**；應只保留長期可重用的結論。
  - **shared/manual.md 不會被 compact-memory workflow 覆寫**，因此它適合作為人工維護的穩定記憶層。
  - **daily snapshots 主要是短期觀測資料**，若某件事只在最新一天短暫出現，不能直接寫成穩定規則。
  - 當資訊不足或彼此矛盾時，必須明確標示不確定性，而不是替主人腦補結論。
- 目前最大的實際限制是：**缺乏近期 issue 資料**，導致無法可靠推導主人在生活、工作、技術選型或協作流程上的長期偏好。

## Open Loops
- 等待下一輪有實質內容的 issue / comment 出現，才能補齊：
  - repo 的實際用途與主要工作流
  - 主人的穩定偏好與協作習慣
  - 已做成的長期技術決策
  - 尚未完成但持續追蹤的事項
- 若後續仍持續沒有 issue，需確認：
  - 是 repo 近期確實沒有活動，
  - 還是 memory 擷取流程沒有抓到資料。
- 目前沒有其他可驗證的未完成事項；唯一明確的 open loop 是：**等待新的可用 issue 更新後再整理記憶**。
