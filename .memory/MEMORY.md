# Repository Memory

## Stable Context
- 🦞 這個 repository 目前可觀測到的長期脈絡非常稀薄；近一段時間的 issue 記憶幾乎只圍繞 **#1**，而它已被確認為**缺乏需求描述與上下文的占位／測試性條目**，並已於 **2026-05-15** 關閉。
- 🦞 龍蝦群對此 repo 的穩定共識是：**沒有足夠背景的 issue，不應被當成可延續的工程任務**。若缺少目標、範圍、驗收條件或討論脈絡，就只做歸檔，不做補腦。
- 🦞 目前看不出明確的產品方向、技術主線、協作節奏或持續中的多工工作流；這不是代表一定沒有工作，而是代表**現有 issue 記憶不足以承載這些結論**。
- 🦞 對主人而言，這個 repo 當前更像是一個**需要先維持追蹤品質**的空殼或早期工作區，而不是已經有穩定開發脈絡的專案。

## Recent Themes
- 🦞 最近反覆出現的主題不是功能推進，而是**記憶整理與議題衛生**：龍蝦群持續確認哪些 issue 其實沒有內容、不能承接、應該停止追蹤。
- 🦞 **#1 的收束** 是近期唯一穩定重複出現的事件：它被多次判定為空白占位，不再視為有效待辦，也不應延伸出任何假設性的需求。
- 🦞 **避免過度推論** 是近期很強的工作原則。面對資料不足，龍蝦群選擇誠實記錄「不知道」，而不是把空白 issue 包裝成已知任務。
- 🦞 多份 daily snapshot 都暗示一件事：**repo 的可用上下文來源不足**。若主人有其他實際工作在進行，可能沒有被 issue 化，或外部對話（例如 Telegram）沒有匯入到可追蹤記憶中。
- 🦞 因此，近期主題可總結為：**清掉無效追蹤項，保留最小可信記憶，等待真正有內容的工作重新進場**。

## Constraints
- 🦞 **資訊不足是目前最大的限制**：現有資料無法支持對功能、架構、除錯、文件或生活安排做出可靠推論。
- 🦞 **issue 關閉狀態被當作重要歸檔訊號**；對像 #1 這種已關閉且無內容的條目，預設不再主動追蹤。
- 🦞 **不可從空白 issue 反推需求**。若沒有明確描述、留言、背景或驗收條件，龍蝦群應維持保守，不建立虛構任務。
- 🦞 **外部脈絡可能缺失**：若需求原本存在於 Telegram 或其他對話管道，但未回填到 repo，可追蹤記憶就會斷裂。
- 🦞 **不確定性需明講**：目前沒有足夠證據證明 repo 完全沒有進行中的工作，只能說在這批 memory 中**看不到可驗證的進展**。

## Open Loops
- 🦞 **#1 若要復活，仍缺核心定義**：
  - 明確目標
  - 背景脈絡
  - 範圍界定
  - 預期輸出或驗收條件
- 🦞 **是否存在未被記錄的實際工作** 仍不確定：可能有其他進行中的事項未進入 issue、未被摘要，或只存在外部對話中。
- 🦞 **repo 的長期上下文仍待建立**：目前還沒有足夠材料沉澱出穩定的技術決策、開發慣例或產品方向。
- 🦞 若主人之後要讓這個 repo 變得可承接，較合理的下一步不是沿用空白殼，而是：
  - 補齊既有 issue 的背景；或
  - 直接建立新的、描述完整的 issue，讓龍蝦群能從可信上下文開始記憶。
