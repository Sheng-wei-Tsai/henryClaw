# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定脈絡很少：這個 repo 的記憶整理流程依賴 **GitHub issue / comment** 作為原始訊號來源，daily snapshots 只是彙整結果，不是事實本體。
- `.memory/shared/manual.md` 是保留給人類手動維護的長期記憶區，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- compact-memory workflow 會讀取手動筆記，但**不會覆寫**它；因此人工維護內容應視為長期基準之一。
- 在目前可見的時間窗內，repo 沒有提供足夠 issue 活動讓龍蝦群萃取出更具體的產品、技術棧、工作流程或主人偏好；這些資訊目前屬於**未知**，不宜臆測。

## Recent Themes
- 最近一整段整理視窗（2026-04-29 至 2026-05-05）都呈現相同狀態：**沒有可用 issue**。
- 因為缺乏活躍 issue，近期沒有形成可辨識的跨議題主題，也沒有新的跨 issue 決策可供沉澱。
- 這代表目前的「近期主題」其實是：**記憶來源暫時中斷，系統處於等待新訊號的觀察期**。

## Constraints
- 記憶整理必須避免把 daily snapshots 當成原始日誌逐段複製；應優先蒸餾成可重用的長期 context。
- 若某件事只短暫出現、且尚未證明是穩定事實，應放在近期主題或 open loop，而不是寫成長期規則。
- 目前最大的限制是**資料不足**：
  - 沒有可用 issue
  - 沒有 labels
  - 沒有跨 issue 決策
  - 沒有可驗證的長期工作模式
- 因此任何關於主人生活、工作脈絡、習慣、技術偏好或未完成專案的具體描述，現在都缺乏證據支撐。
- 若後續資料與目前空白狀態衝突，應以新的 issue / comment 原始內容為準重新整理。

## Open Loops
- 等待下一輪有實質內容的 issue 更新，讓龍蝦群能重新建立 repo 的長期記憶。
- 待補強的關鍵未知項目包括：
  - 這個 repo 的主要用途與目標
  - 主人正在處理的核心工作流
  - 穩定的工程慣例與偏好
  - 已知限制、風險與長期未完成事項
- 若人類已知有穩定規則但近期 issue 沒有反映，建議直接補寫到 `.memory/shared/manual.md`，避免長期記憶持續空白。
