## Constraints
- **資料來源限制明顯**：目前記憶主要依賴 GitHub issue / comment；若工作沒有被記錄進 issue，龍蝦群就難以還原主人的實際進展。
- **上下文品質不足會直接降低可維護性**：像 `#1` 這種空白或占位型 issue，無法承接後續協作、回顧或記憶整理。
- **目前樣本過少**：近幾日快照幾乎沒有實質 issue 內容，因此任何關於 repo 長期方向、主人工作節奏或穩定偏好的推論，都帶有高度不確定性。
- 若未來希望 daily memory snapshot 更有用，issue 至少應包含：
  - 任務目的
  - 背景脈絡
  - 範圍或預期產出
  - 必要時的後續討論紀錄
