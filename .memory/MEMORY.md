# Repository Memory

## Stable Context
- 龍蝦群目前能確認的長期穩定事實很少；這個 repository 在近一週可見的工作脈絡非常稀薄，**唯一持續存在的追蹤節點是 issue `#1`**。
- `#1` 已建立且持續為 **open**，並且曾被指派給**主人與 Copilot**。這表示主人接受一種工作方式：**先建立協作入口，再視情況補齊內容**。
- 目前看起來，這個 repo 的記憶治理原則偏保守：**在需求與上下文不足時，不替主人腦補、不提前展開實作**。這是近多日反覆一致的判讀，可視為暫時穩定的協作習慣。
- 共享手動筆記 `shared/manual.md` 的角色明確：它應保存**穩定規則、長期決策、常見限制與共同習慣**，而不是複製 issue 原文或充當日誌索引。

## Recent Themes
- 近幾天反覆浮出的主題只有一個，而且很明確：**最大的阻塞不是技術，而是任務定義缺失**。
- 龍蝦們多次確認：
  - `#1` 是一個已存在的工作容器；
  - 但缺少標題語意、背景、需求來源、預期成果、完成條件與下一步；
  - 因此目前只能把它當成**保留中的掛載點**，不能當成可直接施工的工程任務。
- 另一個近期反覆出現的訊號是：**人機協作意圖存在，但協作內容尚未落地**。主人已把入口留給自己與 Copilot，表示不是純私人備忘；只是上下文還沒被外化到足以讓代理接手。
- 目前沒有看到其他 issue、標籤、討論串或實作紀錄能補強脈絡，因此整體狀態更像是：
  - 先插旗；
  - 先保留位置；
  - 等主人之後補內容；
  - 再決定是否拆解、排序或執行。

## Constraints
- **資訊來源受限**：目前可採信的內容幾乎只有 issue metadata；沒有足夠的 issue 內文、留言、標籤或外部對話摘要。
- **不可過度推論**：在 `#1` 未補齊背景前，不能可靠判定它是功能開發、文件整理、流程優化、生活提醒、測試用途，或單純占位。
- **不可直接進入實作**：缺少目標、範圍、驗收標準與限制條件時，任何工程拆解都容易偏航。
- **repo 脈絡不足**：近 30 天可見資料幾乎只圍繞 `#1`，因此目前無法建立更完整的產品、技術或生活工作節奏模型。
- **存在不確定性**：若主人曾在 Telegram、口頭對話或其他外部渠道討論過 `#1`，那些脈絡尚未回填到 repo 記憶中；龍蝦群目前看不到，也不應假裝看得到。

## Open Loops
- **`#1` 的真正目標是什麼？**
  - 需要補上問題描述、背景來源、預期成果。
- **`#1` 屬於哪一類工作？**
  - 目前無法確定是工程任務、研究整理、流程事項、生活提醒，還是測試／占位 issue。
- **`#1` 的完成標準與下一步是什麼？**
  - 需要明確交付物、驗收條件、優先順序與責任邊界。
- **是否有外部脈絡尚未匯入？**
  - 若有 Telegram、口頭需求或其他討論，應整理成摘要回填，否則這條支線會持續停在「存在但不可執行」。
- **是否需要拆解成子任務？**
  - 若 `#1` 其實承載的是較大的方向，後續可能需要拆成更小的 issue；但目前資訊不足，尚不能決定。
