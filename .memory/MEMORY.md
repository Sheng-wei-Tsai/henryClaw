# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定脈絡很少：這個 repo 的長期記憶來源分成兩層，**GitHub issue / comment 是原始事實來源**，而 `.memory` 內的內容是整理後的長期記憶。
- `shared/manual.md` 是**人工維護的長期記憶區**，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- 目前提供的 daily snapshots 在連續多天內都沒有可用 issue，因此**尚無法從近期活動中提煉出更具體的專案工作脈絡、技術慣例或主人偏好**。
- 因資訊稀少，除了記憶機制本身之外，**尚不能把任何工作流程、產品方向或工程規則寫成穩定事實**。

## Recent Themes
- 最近一段時間（2026-04-27 至 2026-05-03）的共同主題只有一個：**沒有新的 issue 活動可供整理**。
- 每日快照反覆顯示：
  - 沒有可用 issue
  - 沒有可辨識的跨 issue 主題
  - 沒有新的跨 issue 決策
- 因此近期狀態比較像是**記憶維持期**，不是活躍整理期；龍蝦群目前只能守住既有殼內記錄，等待新的工作訊號浮出水面。

## Constraints
- **不可把 daily snapshots 當成長期事實直接堆疊**；它們是當日彙整，不是穩定規則。
- **不可複製完整 issue 原文進記憶**；應以蒸餾後的長期可重用脈絡為主。
- **人工筆記可被 workflow 讀取，但不會被覆寫**；表示手動維護內容應視為高價值、需審慎更新的長期記憶。
- 目前最大的限制是**資料不足**：
  - 沒有 issue 活動
  - 沒有標籤分布
  - 沒有跨議題決策
  - 沒有足夠證據判斷主人近期的工作重心
- 若後續出現新資訊，需特別注意**不要把單日、單次、短暫訊號誤判為長期規則**。

## Open Loops
- 等待下一輪有實際內容的 issue 更新，才能補齊：
  - repo 的實際工作主題
  - 主人近期關注的任務
  - 穩定的工程習慣與限制
  - 尚未完成的長期事項
- 目前不確定：
  - 這個 repo 是處於暫停、低活躍，還是只是最近 30 天剛好沒有被 issue workflow 捕捉到活動。
  - 是否存在重要脈絡尚未寫入 `shared/manual.md`。
- 建議後續一旦有新 issue / comment 出現，優先觀察：
  - 是否形成可重複的工作模式
  - 是否出現明確決策或限制
  - 是否有值得升格到 `Stable Context` 的長期事實
