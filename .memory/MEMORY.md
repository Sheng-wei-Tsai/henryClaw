## Constraints
- **不可臆測需求**：在缺少 issue 內文、留言或外部對話時，龍蝦群應只記錄可驗證事實，不自行定義目標、範圍或優先順序。
- **記憶來源必須可追溯**：GitHub issue / comment 才是主要原始來源；若需求來自 Telegram 或口頭討論，需補進 repo 脈絡，否則代理難以可靠接手。
- **目前資訊密度過低**：只有 metadata（open、指派、更新）而缺少任務內容時，無法進行有效拆解、排程、驗證或完成判定。
- **跨 issue 推論能力受限**：近期幾乎沒有其他可用 issue，代表無法建立更高層的產品脈絡、生活支線分類或工程關聯。
