# Repository Memory

## Stable Context
- 龍蝦群目前能穩定確認的 repo 脈絡很少；`Sheng-wei-Tsai/henryClaw` 在這段觀測期間幾乎沒有可執行的 issue 內容，主要只看到一個開啟中的 **#1** 作為記憶入口。
- **#1 長期呈現占位／記憶容器性質**：它比較像先放好的掛點，用來承接後續協作與記憶，而不是已經定義清楚的任務。
- repo 的協作傾向偏向 **把零散脈絡整理成可接手的濃縮記憶**，而不是保存逐字聊天紀錄。這點雖然仍來自有限資料，但在多日快照中反覆出現，可視為目前最接近穩定的工作方式。
- 目前沒有足夠證據支持任何明確的產品主線、技術模組、功能方向或生活任務分類；若主人已有其他工作進展，現有記憶來源無法可靠重建。

## Recent Themes
- **資訊不足是主要阻塞**：近幾天反覆出現的不是技術難題，而是缺少任務目標、背景、成功條件與來源對話，導致龍蝦群只能盤點缺口，無法真正推進。
- **記憶基礎設施先於實作**：這段時間的工作重心比較像是在確認「記憶槽位是否存在、能否承接後續脈絡」，而不是交付功能或修 bug。
- **避免誤判、避免腦補**：龍蝦群多次一致採取保守策略，不替 #1 自行定義需求、角色、優先順序或解法。
- **近期最新狀態更安靜**：到 2026-04-25 的整理視窗甚至沒有可用 issue 可供彙整，表示目前可觀測活動非常稀薄；這可能是資料來源暫時空白，也可能是實際上沒有新進展，存在不確定性。

## Constraints
- **不可臆測需求**：在缺少 issue 內文、留言、Telegram 對話或背景資料時，不應自行拆解任務或推定產品方向。
- **資訊來源受限於 issue / comment 與匯入脈絡**：若沒有補進對話紀錄或需求背景，龍蝦群只能記住 metadata 層級事實，無法可靠延續決策歷史。
- **目前不適合估工、排程或設定優先級**：因為 #1 的完成條件、影響範圍與角色定位都未定。
- **shared manual notes 是人工維護的長期記憶區**：它應保留穩定規則、長期決策、常見限制與 repo 習慣，不應複製原始 issue 內容。
- **資料稀薄且可能不完整**：若主人在 repo 外或未被快照捕捉的地方有進展，現有記憶不能假裝知道；需要明確承認不確定性。
