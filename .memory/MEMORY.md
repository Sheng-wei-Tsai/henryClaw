# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定背景很少：這個 repo 的記憶流程依賴 **GitHub issue / comment 作為原始事實來源**，memory 檔案應該只保留整理後的長期脈絡，而不是逐字備份原始討論。
- `.memory/shared/manual.md` 是給人類手動維護的長期記憶區，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- compact-memory workflow 會讀取手動筆記，但**不會覆寫**手動筆記；因此手動筆記可視為較穩定的人類維護層。
- 目前提供的 daily snapshots 中，**沒有可用 issue 活動**，因此尚無法從近期資料萃取出更具體的專案領域知識、工作習慣或技術決策。

## Recent Themes
- 最近一週（2026-05-03 至 2026-05-09）的共同主題非常一致：**沒有新的 issue 可供整理**。
- 在這段期間內：
  - 沒有可辨識的跨 issue 主題
  - 沒有新的跨 issue 決策
  - 沒有標籤分布可供判讀（Top Labels 為 none）
- 因此，近期狀態比較像是**記憶維護空窗期**，而不是專案方向明確變動期。
- 龍蝦群目前只能保守記住：近期沒有足夠訊號可更新長期記憶。

## Constraints
- 記憶整理必須以 **issue / comment** 為主要依據；若沒有 issue 活動，就不應憑空推定專案現況。
- MEMORY 類文件應該是**蒸餾後的長期可重用 context**，不是：
  - 原始日誌
  - issue 索引
  - 每日摘要逐段拼貼
- 若某件事只在單日短暫出現，且尚未形成穩定模式，應放在近期主題或 open loop，而**不應寫成穩定規則**。
- 目前資料量不足，任何關於主人工作流、技術棧、偏好或未完成功能的推論都具有高度不確定性；龍蝦群應維持保守，不過度記憶。

## Open Loops
- 等待下一輪有實際內容的 issue 更新，才能補齊：
  - repo 的實際用途
  - 主人近期在做的工作主題
  - 穩定的工程習慣與決策
  - 真正需要長期追蹤的未完成事項
- 需要後續資料來確認：目前的「無 issue 活動」是暫時空窗，還是此 repo 本來就很少透過 issue 運作。
- 若未來仍持續沒有 issue，可考慮由人類在 `.memory/shared/manual.md` 補充最基本的 repo 背景，否則龍蝦群能保留的長期記憶會持續偏空。
