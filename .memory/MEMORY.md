# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定脈絡很少：這個 repo 的長期記憶來源以 **GitHub issue / comment** 為原始事實基礎，記憶文件只負責整理與蒸餾，不應複製原始討論全文。
- `.memory/shared/manual.md` 是保留給人類手動維護的長期記憶區，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- compact-memory workflow 會讀取手動筆記，但**不會覆寫**它；因此手動筆記可視為較穩定的人類校正層。
- 目前從近一週 daily snapshots 看不出任何穩定的工作主題、開發流程偏好或專案領域知識；這不是代表沒有，而是**目前可用資料不足**。

## Recent Themes
- 最近多日（2026-04-28 至 2026-05-04）都沒有可用 issue 進入整理視窗。
- 因為缺乏新的 issue 活動，近期沒有可辨識的跨議題主題，也沒有新的跨 issue 決策可沉澱。
- 目前 repo 記憶處於一種「低訊號、待補資料」狀態：龍蝦群只能先守住既有框架，等待新的工作痕跡出現。

## Constraints
- 記憶整理必須以**蒸餾後的長期可重用 context** 為主，避免逐段複製 daily snapshot 或 issue 原文。
- 若某件事只在最新一天短暫出現，且尚未證明是穩定事實，應放在近期主題或 open loop，而不是寫成長期規則。
- 當資訊不足或彼此矛盾時，需要明確標示不確定性；就目前資料而言，最大的限制就是：
  - 近一週沒有 issue 可供整理
  - 沒有 labels 可供觀察
  - 無法可靠推定主人的近期工作重心、偏好或未完成技術方向
- 因此，現階段不應憑空補寫 repo 慣例、技術棧或決策歷史。

## Open Loops
- 等待下一輪 issue / comment 更新後，再重新整理可用的長期記憶。
- 需要更多實際工作紀錄，才能判斷：
  - 這個 repo 的主要用途與維護節奏
  - 是否存在固定的工程流程或協作習慣
  - 是否有反覆出現的技術主題、限制或待辦
- 若人類主人已知有穩定規則但尚未寫入，建議補充到 `.memory/shared/manual.md`，讓龍蝦群之後能更穩地接手保管。
