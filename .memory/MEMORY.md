# Repository Memory

## Stable Context
- 龍蝦群目前能確認的長期脈絡很少：這個 repository 在最近可見記憶中，**幾乎沒有穩定展開中的工程主線**，issue 資料量也非常稀薄。
- 目前唯一被反覆提到的案例是 **Issue #1**，但它已被判定為**占位／測試性質的空白 issue**，且已於 **2026-05-15** 關閉；它**不應再被視為有效需求來源**。
- 從多日整理結果可蒸餾出的 repo 工作習慣是：**若 issue 缺乏背景、目標、討論與驗收條件，就不應把它當成可執行任務持續追蹤**。
- 另一個相對穩定的訊號是：主人／龍蝦群傾向維護 **issue 衛生**，寧可關閉沒有內容承載力的紀錄，也不要讓空殼任務長期佔據注意力。
- 目前**無法可靠推定**此 repo 的技術棧、產品方向、部署方式、協作流程細節或外部整合習慣；現有記憶不足以形成這些穩定結論。

## Recent Themes
- 最近幾天的主題不是功能開發，而是**記憶清艙與追蹤系統校正**：把沒有實質內容的 issue 收束，避免後續誤判優先序。
- **Issue #1 的關閉**是近期唯一明確事件。它反覆被確認為：
  - 沒有明確需求
  - 沒有補充討論
  - 沒有可執行規格
  - 沒有可追溯的實作或外部脈絡
- 近期反覆浮出的工作訊號是：**需求若沒有寫清楚，就不要假裝它正在推進**。這比較像流程整理，而不是產品推進。

## Constraints
- **資訊極度不足**：目前記憶來源幾乎只剩一個已關閉且內容空白的 issue，無法支撐更深入的工程判讀。
- **不可從 #1 延伸推論需求**：既然它已被多次確認為占位／測試性紀錄，就不應再從中腦補功能、規格或待辦。
- **若工作未進入 issue 脈絡，龍蝦群就難以接手**：現有資料明確顯示，缺少背景與上下文會讓後續整理、協作與追蹤失效。
- **目前沒有可靠證據**顯示是否存在 issue 之外的外部討論（如 Telegram）、程式碼分支、隱性任務或生活安排；若有，現階段都屬未記錄狀態。
- shared manual notes 目前只有維護原則，**沒有額外的人類手動長期記憶可補強**。

## Open Loops
- 若主人之後要在這個 repo 重新啟動工作，**需要以新的 issue 明確定義任務**，至少補上：
  - 背景脈絡
  - 目標
  - 預期成果
  - 驗收條件
- 一個持續未解的問題是：**這個 repo 是否其實有真實工作在進行，只是沒有被記錄到 issue 系統**。目前無法確認。
- 可考慮建立更一致的 **issue 模板**，避免再次出現只有標題、沒有內容的空殼紀錄。
- 目前沒有其他可確認的未完成工程事項；若有新主題出現，應先觀察它是否跨日持續、是否具備足夠上下文，再決定是否升格為 Stable Context。
