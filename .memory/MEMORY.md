# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定脈絡很少：這個 repo 的記憶整理流程依賴 **GitHub issue / comment** 作為原始資料來源，memory 檔案應該是整理後的長期記憶，而不是逐字備份日誌。
- `.memory/shared/manual.md` 是給人類手動維護的長期記憶區，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- compact-memory workflow 會讀取手動筆記，但**不會覆寫**手動筆記；因此人工維護內容應視為可長期保留的高信號上下文。
- 目前從近幾日 snapshots 看不出任何穩定的專案主題、工作流細節或技術決策；這不是代表不存在，而是目前可用資料不足。

## Constraints
- 記憶整理必須以 **issue / comment** 為主要事實來源；若沒有 issue 活動，就不應憑空推定專案現況。
- MEMORY 類文件應該做**長期可重用的蒸餾**，避免複製原始 issue 內容或逐日快照。
- 若某件事只短暫出現、尚未形成穩定模式，應放在近期主題或 open loop，而不是寫成固定規則。
- 目前資料明顯不足，因此任何關於主人工作習慣、技術棧、優先事項的推論都應保守處理。
- 現階段沒有 labels、沒有跨 issue 訊號，代表自動整理能提供的 repo context 非常有限。
