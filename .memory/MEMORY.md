# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定背景很少：這個 repo 的記憶來源以 `.memory` 目錄為主，並且應由長期脈絡整理而成，而不是逐日事件抄錄。
- `shared/manual.md` 的角色明確：它是給人類手動維護的長期記憶區，適合放穩定規則、長期決策、常見限制，以及 agent 共同遵守的 repo 習慣。
- GitHub issue / comment 被視為原始事實來源；記憶文件應該做提煉，不應複製完整 issue 原文。
- `compact-memory workflow` 會讀取手動筆記，但不會覆寫它；因此手動維護內容可視為較穩定、需要被保留的記憶層。

## Recent Themes
- 最近這一段時間（至少 2026-05-02 到 2026-05-08），每日快照都顯示：**沒有可用 issue**。
- 因為缺乏新的 issue 活動，目前**沒有可辨識的跨 issue 主題**，也**沒有新的跨 issue 決策**可供蒸餾。
- 這代表目前的主要狀態不是「有一串正在收斂的工作」，而是「記憶整理流程持續運作，但沒有新材料進來」。

## Constraints
- 記憶整理必須以「長期可重用」為優先，避免把 daily snapshot 當成原始日誌逐段搬運。
- 若某件事只在最新一天短暫出現、尚未形成穩定模式，應放在近期主題或 open loop，而不應寫成穩定規則。
- 目前資料量不足，無法可靠推定：
  - repo 的實際技術棧
  - 主人的工作偏好
  - 常見開發流程
  - 任何具體產品方向
- 因此龍蝦群需要保守：**沒有證據就不要假裝記得**。

## Open Loops
- 等待下一輪 issue 更新後，再判斷是否出現可蒸餾的長期脈絡。
- 目前缺少足夠訊號來建立更具體的 repo context；若之後仍長期沒有 issue，可能需要確認：
  - 這個 repo 是否處於低活躍或封存狀態
  - 是否有其他記憶來源尚未納入 `.memory`
  - 是否需要由人類在 `shared/manual.md` 補入基礎背景
- 不確定性：現有資料只反覆說明「沒有新 issue」，**不能據此推論 repo 沒有工作、沒有需求，或沒有歷史脈絡**；只能確認在這段觀測窗內，記憶系統沒有收到新的 issue-based 訊號。
