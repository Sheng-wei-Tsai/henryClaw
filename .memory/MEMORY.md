# Repository Memory

## Stable Context
- 龍蝦群目前能確認的長期脈絡很少；這個 repository 在最近可見記憶中，**沒有穩定且持續推進的功能主線、產品方向或工程模組脈絡**。
- 目前唯一被明確整理過的 issue 是 **#1**，而它已被判定為**占位／測試性紀錄**，不承載可執行需求，也**不應再被視為正式任務主線**。
- 從這次整理可蒸餾出一條可重用的協作習慣：**若 issue 缺少背景、目標、驗收條件與討論脈絡，龍蝦群不應自行腦補成工程工作**。
- 另一條較穩定的工作原則是：**維護 issue 衛生本身就是工作的一部分**。空白或無內容的追蹤項，應及早辨識、關閉或改以更清楚的新 issue 重建。

## Recent Themes
- 最近幾天的主旋律不是開發，而是**清理追蹤噪音**。龍蝦們反覆確認：#1 沒有實質內容，因此在 2026-05-15 關閉是合理收束。
- 近期反覆浮現的訊號是：**資訊不足時先停止延伸，不硬做、不硬推論**。這代表主人目前至少接受一種較保守的記憶整理方式：先確保紀錄品質，再談後續執行。
- 另一個近期主題是**記錄品質不足**。從 #1 的狀況看，若只有標題或空殼紀錄，下一隻龍蝦幾乎無法接手，也無法判斷是否與外部對話（如 Telegram）有關。
- 目前也反覆出現一個空白訊號：**看不到其他正在進行中的主線任務**。但這不一定代表主人沒有在做事，更可能是工作尚未進入 issue 記錄系統。

## Constraints
- **資料來源極少**：近幾日可用內容幾乎只有 #1，且資訊非常稀薄，因此很多判斷只能停留在「目前看不到」，不能升格為強結論。
- **不可從空白 issue 推導需求**：沒有描述、留言、外部連結或驗收標準時，龍蝦群應避免自行補完需求。
- **#1 不適合復用**：若未來真的有對應想法，較合理的做法是**另開新 issue**，而不是回頭沿用這個已關閉且缺乏上下文的殼。
- **外部脈絡缺失**：若任務曾在 Telegram 或其他地方討論，但沒有回填摘要或連結，repository 內的記憶會斷裂，造成接手困難。
- **不確定性高**：目前無法確認這個 repo 是否只是暫時沉寂、是否有未記錄中的實際工作，或是否存在 issue 之外的主要協作流程。

## Open Loops
- 是否需要建立或補強 **issue 模板**，至少要求：
  - 背景脈絡
  - 明確目標
  - 預期成果或驗收條件
  - 外部討論連結／摘要
- 若主人其實有正在推進的工作，**需要補進 repository 可追蹤的紀錄**；否則龍蝦群目前無法替主人維持長期記憶網。
- **#1 背後是否曾對應真實想法** 仍不明。如果有，尚待以新 issue 重新描述，而不是從舊殼回推。
- 目前沒有可確認的活躍主線；後續需觀察這是**暫時沒有任務**，還是**任務存在但未被記錄**。
