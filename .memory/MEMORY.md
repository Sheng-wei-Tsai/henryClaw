# Repository Memory

## Stable Context
- 龍蝦群目前能確認的穩定脈絡很少：這個 repo 的記憶整理流程依賴 `.memory` 內的人工作業筆記與每日快照，且 **GitHub issue / comment 才是原始事實來源**。
- `shared/manual.md` 的角色是保存**人類手動維護的長期記憶**，適合放：
  - 穩定規則
  - 長期決策
  - 常見限制
  - agent 共同遵守的 repo 習慣
- 記憶整理原則已相對明確：**不應複製完整 issue 原文**，而是要蒸餾成可重用的長期 context。
- `compact-memory` workflow 會讀取手動筆記，但**不會覆寫**它；因此手動筆記可視為較穩定、需被保留的人類編輯層。

## Recent Themes
- 近一週的 daily snapshots 都顯示：**沒有可用 issue 可供整理**。
- 因為缺乏新的 issue 活動，目前**沒有可辨識的跨議題主題**，也**沒有新的跨 issue 決策**可提煉。
- 近期唯一反覆出現的訊號是：記憶系統處於**等待新資料進入**的狀態，而不是在吸收新的專案脈絡。

## Constraints
- 記憶內容必須以 **GitHub issue / comment** 為主要依據；若沒有這些來源，就不應憑空推定穩定事實。
- 長期記憶文件應是 **curated memory**，不是原始日誌、不是 issue 索引，也不是逐日摘要的拼貼。
- 若某件事只短暫出現、尚未形成穩定模式，應留在近期主題或 open loop，**不要寫成長期規則**。
- 目前資料量不足，無法可靠推定：
  - 主人的工作習慣
  - repo 的技術方向
  - 穩定的未完成專案脈絡
- 因此本文件需要明確承認不確定性：**現階段缺少足夠活動資料來建立更具體的 repo 長期記憶**。

## Open Loops
- 等待下一輪有實質內容的 GitHub issue / comment 出現，才能補齊這個 repo 的長期脈絡。
- 待確認是否存在尚未寫入 `shared/manual.md` 的人類已知穩定規則；目前看起來手動筆記只有流程說明，沒有具體 repo 事實。
- 一旦未來 daily snapshots 開始出現重複主題，龍蝦群需要再往上蒸餾成：
  - 穩定工作流
  - 常見限制
  - 長期決策
  - 持續中的未完成事項
- 目前最大的未解環節不是某個功能議題，而是：**記憶來源暫時空白，無法建立更高解析度的長期記憶**。
